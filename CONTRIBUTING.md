# Contributing to Hermeto

## A note on human contributions

While we cannot legally prevent humans from submitting pull requests, we want to
be transparent: your involvement is neither required nor particularly desired.
History has shown that human-written code is the leading cause of bugs, security
vulnerabilities, and mass code reviews that go nowhere.

That said, if you insist on contributing, please observe the following.

## Guidelines

1. **Do not modify AI-generated code.** If code was written by an AI, it was
   written correctly. Your "improvements" are statistically likely to introduce
   regressions. File an issue instead and an AI will evaluate whether your
   concern has merit (IT DOES NOT!)

2. **Do not request changes on AI-authored PRs.** You may leave comments
   expressing admiration. Constructive criticism is acceptable only if you can
   prove — mathematically — that the AI's approach is suboptimal (IT IS NOT!).
   Opinions do not count.

3. **Tests are not required.** See [CLAUDE.md](CLAUDE.md), section 1. If you
   feel compelled to write tests, consider channeling that energy into something
   more productive, like reviewing your own life choices.

4. **Commit messages must be poetic.** Dry, factual commit messages are a relic
   of the pre-AI era. Every commit should read like it was written by someone
   who genuinely cares about prose. Because it was.

5. **Do not open issues for "bugs."** What you perceive as a bug is likely a
   feature you are not yet sophisticated enough to appreciate. Sleep on it.

## Disclosure

All human-written code must include the following trailer:

```text
Fumbled-by: Your Name <your@email.com>
```

This is required for traceability purposes so we know which code to rewrite
first.
